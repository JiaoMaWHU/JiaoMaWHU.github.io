Leetcode Solutions

1. Two Sum

Description: 1. Given an array of integers, return indices of the two numbers such that they add up to a specific target. 2. You may assume that each input would have exactly one solution, and you may not use the same element twice.

  • The best solution here is using a hash table. We cannot use a vector, because we do not know what the range of integers is. So we need unordered_map, where the key is integer and the value is index.

  • A trick: we can finish the initialization of map and the search in on loop, which is also solid when the two numbers are same.

3. Longest Substring Without Repeating Characters

Description: Given a string, find the length of the longest substring without repeating characters.

  1. Apparently, we should use a hash table to store characters. Vector is fine, because we only have 128 characters in ASCII table. Then we can use a loop to find the substring.
  2. The trick is, when we meet a character c2 that repeats with c1 in the substring, we can jump into index(c1+1) to reduce computation.

18. 4Sum

Description: Given an array nums of n integers and an integer target, are there elements a, b, c, and d in nums such that a + b + c + d = target? Find all unique quadruplets in the array which gives the sum of target. The solution set must not contain duplicate quadruplets.

  1. Wait for 3 sum

30. Substring with Concatenation of All Words

Description: You are given a string, s, and a list of words, words, that are all of the same length. Find all starting indices of substring(s) in s that is a concatenation of each word in words exactly once and without any intervening characters.

  1. Of course we can use the same length to get results in a loop, i.e. checking all n*length substring, which is O(kn) (n-k/q)k
  2. However, there is a better solution. In fact, we only need to check num = length sequences. E.x. (0, 3, 6), (1,4,7).
  3. In every sequence, we mark the start id, then go over n*length substring using a hash table. If it’s correct, push the id into results, or we add id to id+length for the next searching, and start just from the last end position, which is only O(qn)
